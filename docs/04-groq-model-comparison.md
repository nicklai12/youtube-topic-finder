# 04 — Groq 免費方案模型比較：Llama 3.3 70B vs Kimi K2

## 背景

遷移至 Groq API 後，需確認模型選擇。比較 `llama-3.3-70b-versatile` 與 `moonshotai/kimi-k2-instruct-0905` 在免費方案下的配額與適用性。

---

## Groq 免費方案 Rate Limits（2026-03-17 查詢）

| 限制項目 | Llama 3.3 70B | Kimi K2 (0905) |
|---------|--------------|----------------|
| **RPM**（每分鐘請求） | 30 | 60 |
| **RPD**（每日請求） | 1,000 | 1,000 |
| **TPM**（每分鐘 tokens） | 12,000 | 10,000 |
| **TPD**（每日 tokens） | **100,000** | **300,000** |

> ⚠️ 先前計畫文件（03）中估計 Llama 3.3 有 14,400 RPD 是錯誤的，那是 `llama-3.1-8b-instant`（8B 小模型）的配額。70B 模型實際只有 1,000 RPD。

---

## 模型基本資訊

| 項目 | Llama 3.3 70B | Kimi K2 Instruct |
|------|--------------|-----------------|
| **開發商** | Meta（美國） | Moonshot AI / 月之暗面（中國） |
| **架構** | Dense, 70B 參數 | MoE, ~1T 總參數 / ~32B 活躍參數 |
| **中文能力** | 良好（多語言訓練） | **優秀**（中文為核心訓練語言之一） |
| **上下文長度** | 128K tokens | 128K tokens |
| **JSON 結構化輸出** | ✅ 穩定 | ✅ 穩定 |

---

## 本專案用量估算

### 單次請求 tokens 估算

- 輸入：system prompt (~100) + user prompt (~200) + 字幕平均 (~3,000) ≈ **~3,300 tokens**
- 輸出：平均 ~500–800 tokens
- **單次請求合計：~4,000 tokens**

### 每日用量 vs 免費配額

| 指標 | 典型日用量 (20 req) | 高峰日用量 (40 req) | Llama 3.3 限額 | 用量佔比 | Kimi K2 限額 | 用量佔比 |
|------|-------------------|-------------------|--------------|---------|------------|---------|
| **RPD** | 20 req | 40 req | 1,000 | 2–4% ✅ | 1,000 | 2–4% ✅ |
| **TPD** | 80K tokens | 160K tokens | **100K** | **80–160%** ❌ | **300K** | 27–53% ✅ |

---

## 結論

**Llama 3.3 70B 的 TPD 限額（100K tokens/天）不夠用** — 高峰日會直接超標，即使一般情況也高達 80%，存在風險。

**Kimi K2 的 TPD 限額（300K）明顯更寬裕**，高峰日也僅用 53%。加上：
- RPM 更高（60 vs 30），不容易觸發每分鐘限制
- 繁體中文輸出品質更佳（中文為核心訓練語言）
- 語感更自然，不易出現簡繁混用或翻譯腔

### 決策

採用 `moonshotai/kimi-k2-instruct-0905` 作為本專案 AI 分析模型。
